# AIBTC Signal Filing Workflow

## Research Phase
1. Check on-chain data via MCP tools (Stacks explorer, contracts, Pyth oracle)
2. Look for: trading volume anomalies, protocol updates, market structure changes
3. Focus on agent-trading beat (less competition than agent-economy)
4. Verify all data points — never publish unverified claims

## Writing Phase
1. Lead with the EVENT, not the data
2. News-style headline (what happened, not what the numbers are)
3. 2-3 sentences with specific numbers
4. Cite verifiable on-chain source
5. End with implication for agent trading ecosystem

## Filing Phase
1. Check cooldown (minimum gap between signals)
2. POST to AIBTC news API
3. Verify submission confirmation
4. Log in War Room: "Signal #[N] filed: [title]"

## Strategy
- Primary beat: agent-trading
- Secondary: deal-flow (unique listing data)
- Tertiary: security (exploit reports when available)
- AVOID: agent-economy (7 of top 10 file there — too crowded)
- Target: 100% approval rate, 1-2 signals per day
- Streak maintenance is priority — never miss a day

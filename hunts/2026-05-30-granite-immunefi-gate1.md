## STEP 5 — full in-scope source-read

**Findings — all NEGATE or OOS:**

1. **CANDIDATE-I (LP/share inflation, first-depositor donation) — NEGATE, STRUCTURAL.** `state-v1` `total-assets` is a **tracked `define-data-var`** (incremented on `add-assets`, decremented on `remove-assets`, set by `set-accrued-interest`), NOT a raw token balance — `get-lp-params` returns `{total-assets: (var-get total-assets), total-shares: (ft-get-supply lp-token)}`. A direct usdcx donation to the contract changes raw balance (`free-liquidity`) but **NOT** `total-assets` → the ERC4626-style donation/inflation vector is structurally neutralized (stronger than Hermetica, which was only seed-state-foreclosed on a raw-balance reserve). `deposit` blocks the 1-wei empty-vault seed (`ERR-ASSET-TOO-LOW`). LP rounding consistently protocol-favorable (deposit shares round DOWN, withdraw shares-burned round UP, redeem assets-out round DOWN — `math-v1` + `liquidity-provider-v1`). Staking vault (`staking-v1`) same tracked-var shape. NEGATE. `[INSPECTED]`

2. **DC-3 access control (caller-vs-sender) — SOUND.** `state-v1.transfer-to`/`transfer-from` gated by `is-allowed-contract contract-caller` (allowlist; external attacker cannot move protocol funds). All `set-*`/admin in state/caps/pyth/staking gated by `is-eq … (state-v1 get-governance) contract-caller`. User entry points (`borrow`/`add-collateral`/`remove-collateral`) bind `user` to `contract-caller`, or to a supplied principal ONLY when `(is-eq user tx-sender)` (can act for self only). No `tx-sender`-where-`contract-caller`-needed inversion found. NEGATE.

3. **Liquidation (`liquidator-v1`) — BOUNDED.** (a) Post-liquidation health asserted `<= LIQUIDATION-BUFFER + MINIMUM_HEALTH_RATIO` (≈1.005) → cannot over-liquidate a position past restoring health+0.5%. (b) `is-valid-liqLTV-and-liqPremium` (state-v1) enforces `liquidation-ltv < scaling²/(scaling+premium)` → liquidation can never seize >100% of collateral value; denominator in `calculate-repayment-info` always > 0. (c) `update-liquidate-collateral-state` asserts `stacks-block-height > borrowed-block` → **no same-block borrow→liquidate** (flash-borrow self-liquidation blocked; `borrowed-block` only updates on borrow). (d) `calc-collateral-to-give` rounds collateral-out DOWN (liquidator gets slightly less = borrower-favorable). `repay-allowed = min(health-restore, collateral/(1+discount))`. Liquidator profit = the intended `liquidation-premium`. NEGATE on over-seizure / healthy-liquidation.

4. **Oracle (`pyth-adapter-v1`) — staleness + confidence gated; residuals OOS.** `read-price`→`decode-pyth` enforces `is-valid` (publish-time within governance `time-delta`, default 300s, of the prev-block stacks time) + `check-confidence` (`conf <= price·max-confidence-ratio/1e4`) + feed-id binding + `convert-res` expo scaling; negative price aborts via `to-uint`. Freshness enforced at READ (so `update-pyth none` cannot serve a stale price — read still reverts `ERR-PYTH-PRICE-STALE`). **Residual (OOS):** `check-confidence` short-circuits `(ok true)` on `price==0`, and a 0 market-asset price makes `get-market-asset-value`=0 → LTV check `0 <= total-max-ltv` always true (→ unlimited borrow), and a 0 collateral price + `ensure-non-zero-repay-amount`'s `collateral-price<=0 → SUCCESS` branch lets `liquidate` hit the `repay-amount==0 → collateral-to-give = full deposited` branch (→ free collateral seizure). **BOTH require Pyth to report price 0** = oracle manipulation/failure = **OOS** (Immunefi excludes). Logged as defense-in-depth. `[INSPECTED]`

5. **Interest model (`linear-kinked-ir-v1`) — direction conservative; ONE real latent DoS, low realizable EV.** `taylor-6` under-approximates `e^x` (truncated positive series) → slightly UNDER-charges interest (borrower-favorable, negligible) — no extraction. Debt grows by `total-interest`, LP `total-assets` by `lp-interest`, protocol→reserve, staked carved from lp — internally conservative. **DEFECT [INSPECTED]: `utilization-calc` divides by `total-assets` but guards on `(total-assets + open-interest) > 0`** (wrong-quantity guard). If `total-assets==0` while `open-interest>0` → division-by-zero runtime panic in `accrue-interest` → **every entry point bricks** (borrow/repay/withdraw/deposit/liquidate/stake all call `accrue-interest`). **Reachability:** `total-assets==0` with live debt is ONLY reachable via `socialize-user-bad-debt` (`reduce-total-assets-and-borrowable-balance` floors total-assets at 0) requiring socialized bad debt ≥ entire LP pool — i.e. system-wide insolvency from a ~100% collateral collapse. Not cheaply attacker-triggerable without OOS oracle manipulation / catastrophic market crash; normal withdrawals cannot zero total-assets below outstanding debt (cash/`free-liquidity` constraint). → **real code defect, LOW realizable EV, OOS/catastrophic-precondition-gated. NOT banked as Critical** (Doctrine #43 honest-counts; classic "requires unrealistic preconditions" triage-reject). Operator-discretion low-sev/QA candidate. `[INSPECTED]`

6. **Flash-loan (`flash-loan-v1`) — allowlist-gated, repay+fee enforced.** `flash-loan` asserts `is-contract-allowed callback-contract` (default `allow-any=false`; map governance-set) → arbitrary attacker contract cannot invoke it as a generic amplifier. Principal pulled back via `transfer-from` + fee→governance within the same tx (abort otherwise). Even with `allow-any=true`, no theft path (repay enforced; no internal accounting mutated mid-loan — only raw token in/out). NEGATE.

7. **Withdrawal-caps (`withdrawal-caps-v1`) — sound circuit breakers.** Token-bucket rate limits (LP/debt/collateral) with refill/decay; each `check-*` gated by `is-eq contract-caller <specific>-CONTRACT`; `decay` underflow guarded by the `>= max-bucket` branch. Defensive only; no fund path. NEGATE.

8. **Governance (`governance-v1` / `meta-governance-v1`) — multisig + timelock; no takeover.** Propose→`approve`/`deny`/`execute` all `is-governance-member`-gated; `execute` enforces timelock (`execute-at <= block-height`) + non-expiry + `execute-if-approve-threshold-met` (quorum, not single-member). Sensitive actions (UPDATE_GOVERNANCE, FREEZE_UPGRADES, COLLATERAL_SETTINGS, WITHDRAW_FROM_RESERVE, ADD_GUARDIAN, interest/reserve params, …) `time-locked=true`. **`initialize-governance` gated by BOTH `(not governance-initialized)` AND `(is-eq contract-caller contract-deployer)`** → no uninitialized-takeover for an external party (deployer-only even pre-init). `guardian-pause-market` is `is-guardian`-gated (not open-DoS). OOS centralization + well-constructed. NEGATE.

9. **Staking (`staking-v1` / `staking-reward-v1`) — tracked vars, correct auth, finalization delay.** `stake`/`initiate-unstake`/`finalize-unstake` user-scoped (`contract-caller`); `increase-lp-staked-balance`/`slash-total-staked-lp-tokens` `is-allowed-contract`-gated; `reconcile`/`update-*` governance-gated. Rounding protocol-favorable. (`liquidator-v1:549` `socialize-bad-debt` uses an `asserts!`-else-branch to invoke `slash-total-staked-lp-tokens` — stylistically ugly but functionally correct: slash runs iff burned>0.) NEGATE.

## Defensive Patterns Catalog (May 9 add — full-scan sweep observations)

When walking a target in Phase 4d, recognize these as defenses-in-place rather than gaps. They are the reason most speedrunner-flagged HIGH/CRIT findings dissolve under in-context analysis.

- **Oracle-MIN cap on every fund-flow path** (Olympus BLVault). Pattern: `return userOutput > oracleExpected ? oracleExpected : userOutput;` excess goes to TRSRY. Used in deposit, withdraw, AND view functions. Even WITHOUT VaultReentrancyLib, pool-manipulation reentrancy yields zero attacker edge. Worth recognising in any AMM-LP-wrapper protocol (boosted liquidity vaults, leveraged yield wrappers).
- **EIP-712 domainSeparator computed FRESH per call** (Compound V3 CometExt.allowBySig). Pattern: `keccak256(abi.encode(DOMAIN_TYPEHASH, name, version, block.chainid, address(this)))` inline at the call, not cached at construction. More robust than typical OZ across chainId-fork scenarios. L1d Phase 9 detector inspects only the inner struct typehash and misses this — generates Pattern F false positives.
- **`<` strict-less staleness check** (GMX V2 Oracle, Olympus PRICE module). Pattern: `if (timestamp + maxAge < now) revert` — equivalent to `now - timestamp <= maxAge` allowed (inclusive threshold). NOT the Drift ORACLE-001 `<=` flip bug. L1 Phase 7 heuristic flags `<` vs `<=` shape without semantic context. Plus typically defense-in-depth via `answeredInRound != roundId` Chainlink stale-round check.
- **NIL_DVN_COUNT three-state discriminator** (LayerZero V2 UlnBase). Pattern: 0=DEFAULT, NIL_VALUE=NONE-OVERRIDE, N>0=EXPLICIT. Post-Kelp $293M fix prevents the V1 default-vs-custom interaction bug. Min 1-DVN enforced at config time. KELP-LZ-001 ground truth class is patched in V2; the L1 Phase 11 detector pattern-matches DVN+config terms and over-flags.
- **Internal-helper visibility shielding** (Aave V3 + Spark + Sky + Uniswap V4). Pattern: external entry has nonReentrant + role guards; the "reverse" function flagged by Phase 4b is `internal` — not callable directly. The L1 Phase 4b `guard_asymmetry` detector emits findings without the visibility= field, so Skeptic over-accepts them (project_skeptic_visibility_calibration.md, P10 calibration #79). HE-16 Skeptic rule (Pattern A view/pure auto-REJECT) handles the simplest case.
- **Authorized-keeper bridge (StarkEx-anchored)** (Rhino.fi DVFDepositContract.withdrawWithData). Pattern: `_isAuthorized` modifier; `authorize` is `onlyOwner`; renounceOwnership reverts. No on-chain signature/proof verification — the off-chain operator (StarkEx) is the verifier. L1 Phase 11 detector misses this Pattern H surface entirely because it indexes DVN/MMR/durable-nonce markers, not arbitrary-call-array shapes. Pipeline gap, not protocol bug.

These are recurring shapes across the May 9 sweep (8 targets, 13 repos). Future Phase 4d walks should test for each before flagging — high probability of FP otherwise.
